Introduction

The Project

Azure Audio is a cross-platform audio library for games being written in C with the mission to bring powerful, feature-full 3D audio to developers of various backgrounds in audio. It is meant to be a simple, extensible, easy-to-use API that you can include in your projects for commercial or non-commercial use. You are encouraged to write your own modifications for any new features you may desire, or you may request a feature by contacting me. I do ask that, should you make any modifications (improvements, additions, etc.), that you consider sending them to me so I can add them to the core library, but you are in no way required to do so. The only condition to using this library is attribution to the original author, as detailed in the license.

On top of this will be a C++ wrapper library that will act as a high-level abstraction library designed to make use of the library as clean and easy as possible while maintaining control over how it works. With this, many default settings will be provided to make practical use very simple, while still allowing lower-level access whenever necessary or desired.

Naming Conventions

Based on my experience and personal preferences, I've chosen these naming conventions because they are distinct for each type of name they represent, which not only helps avoid conflicts, but also provides a way to identify what something is without looking it up or tagging it with an identifier. They are meant to be a good balance between clarity, and brevity.

Preprocessor Macros

ALL_CAPS_WITH_UNDERSCORES
Prefixed with AZA_ for the vast majority. The prefix AZURE_AUDIO_ is used exclusively for parameters the user may wish to change for compilation, and affect the build process.

Functions

azaCapitalizeEveryWordWithoutSpaces
Always prefixed with aza.

Data Structures Repeatedly Passed to Functions

aza<FunctionName>Data

Examples:
  1. AZA_PREPROCESSOR_VALUE
  2. AZURE_AUDIO_COMPILE_SETTING
  3. azaFoo( ... , azaFooData *data )

Error Handling

Any library function you call will return an error status as an int. On success, it will always return AZA_SUCCESS. Simple error checking can be done this way. You can store the value returned by the function, or, alternatively, can check the last error code by using azaGetError().

Reference

azaInit()

  1. int azaInit(void)
Description:

Initializes the library. This must be called before any other library functions! If this is omitted, behavior is undefined.

Returns:

AZA_SUCCESS on success

azaGetError()

  1. int azaGetError(void)
Description:

Gets the last error code set.

Returns:

Any library-specified error code.

Error Codes

These are all the possible error codes.

  1. #define AZA_SUCCESS 0
  2. // The operation completed successfully
  3. #define AZA_ERROR_NULL_POINTER 1
  4. // A pointer was unexpectedly null
  5. #define AZA_ERROR_PORTAUDIO 2
  6. // PortAudio has generated an error
